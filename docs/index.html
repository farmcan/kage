两条命令，讲清 KAGE

上下文不该困在一个 Agent 里。

x 代表 Codex，c 代表 Claude。箭头左边是来源，右边是目标。你选择会话，KAGE 生成目标 Agent 能直接恢复的原生文件。

kage x2x Codex 分身

复制上下文，不复制限制。

从当前 Codex 会话分出一个独立会话。原会话保留，新分身可以验证另一种方案、并行推进或安全试错。

不是导出一段文本

而是创建一个能继续工作的会话。

KAGE 读取 Agent 已经写在本机的记录，转换成目标工具认识的格式，并保留来源关系。整个过程默认不上传 transcript。

01 原生可恢复

生成 Codex、Claude Code 或 QoderCLI 能直接 resume 的会话文件。

02 来源可追溯

lineage sidecar 记录父会话、子会话和转换路径，分身不会失去来处。

03 默认只在本机

没有托管索引、远程同步或 transcript 上传，现有会话留在磁盘。

04 启动由你决定

KAGE 只准备文件和 resume 命令，不会在转换后自动启动 Agent。

学会一次，就会所有路由

来源 2 目标。

c2x 就是 Claude → Codex，x2x 就是 Codex → Codex。相同语法也能连接 QoderCLI。

x Codex c Claude q QoderCLI
kage x2x 创建 Codex 分身，保留一条独立工作线。
kage c2x 让 Claude Code 的会话在 Codex 里继续。
kage x2c 把 Codex 上下文交给 Claude Code。
kage x2q / q2x 在 Codex 与 QoderCLI 之间转换会话。
kage sessions 按项目浏览本机已有的 Agent 会话。
kage search "auth" 搜索标题、提示词和 transcript 文本。
