# KAGE v0.1.0

KAGE v0.1.0 is the first desktop-focused release of KAGE: a local-first workspace for finding, searching, continuing, and moving AI coding sessions across Claude Code, Codex, and QoderCLI.

## What You Can Do

- Browse local sessions by project directory.
- Filter by agent and include subdirectories when scanning a workspace.
- Search transcript text from the desktop app or CLI.
- Continue a selected session inside the KAGE embedded terminal.
- Bridge a useful session into another agent's native session format.
- Replay a session story for review.
- Package a local unsigned macOS DMG with `app/package.sh`.

## Try It

Download the unsigned macOS DMG:

```text
https://github.com/farmcan/kage/releases/download/v0.1.0/KAGE-0.1.0.dmg
```

If Gatekeeper blocks the first launch, right-click `KAGE.app`, choose `Open`, then confirm.

Install the CLI:

```bash
curl -fsSL https://raw.githubusercontent.com/farmcan/kage/main/install.sh | bash
```

Check local readiness:

```bash
kage doctor
kage sessions
```

Build and open the desktop app from a checkout:

```bash
swift build --package-path app
(cd app && ./bundle.sh)
open app/.build/release/KAGE.app
```

Package an unsigned DMG:

```bash
./app/package.sh
open app/.build/release/KAGE-0.1.0.dmg
```

## Known Limitations

- The macOS app is unsigned unless you provide `KAGE_CODESIGN_IDENTITY`.
- Notarization requires `KAGE_NOTARY_PROFILE`.
- The embedded terminal is intentionally session-scoped, not a general terminal replacement.
- Mobile and multi-device views are planned, but not part of this release.
- GitHub release automation still needs a token with `workflow` scope before the workflow can be added.
