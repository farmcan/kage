Why people star KAGE

Most agent tools focus on the current run. KAGE focuses on what happens after useful work is already buried in local transcripts: finding it again, reviewing it, branching it, and moving it across agents.

Stop losing useful context

Turn scattered Claude Code, Codex, QoderCLI, and QoderWork sessions into reusable project memory.

Keep memory local

Search and act on local transcript stores without sending session content to a hosted index.

Bridge instead of paste

Move useful context into another agent's native resume format instead of hand-copying transcripts.

Watch an active project

Desktop, CLI, local web, task board, and release work are all moving in the same open repo.

Current desktop surface

The current product surface is the local web/desktop workspace: recent local sessions, conversation bubbles, explicit send controls, task board status, and a clear path back to Terminal.app when a noisy session needs it.

Workflows that match local agent memory

KAGE is not another coding agent. It is the local memory layer around the agents you already use: useful past context becomes searchable, reviewable, branchable, and portable.

Dispatch board

Start a one-off Claude Code, Codex, or QoderCLI task and track it from queued to review.

New Session

Start Codex, Claude Code, or QoderCLI in the watched project from KAGE or Terminal.app.

Fork / Bridge

Branch a session or move useful context into another agent's native resume format.

Search / Replay

Find old local work by project or transcript text, then replay it as a local review artifact.

Fit check

Use KAGE if

  • You use more than one AI coding agent.
  • You need to find old local sessions by project or transcript text.
  • You want a local board to dispatch, review, bridge, and replay agent work.

Skip it for now if

  • You only use one agent and never revisit old sessions.
  • You need a signed and notarized macOS installer today.
  • You expect hosted sync, team sharing, or a public remote relay today.

Mobile agent monitor without becoming cloud-first

kage serve opens the same local memory layer as a responsive monitor for trusted LAN use. See what your agents are doing from your phone, inspect live or completed sessions, and keep dispatch as a secondary action without moving transcript content off disk.

Agent monitor

Check active sessions and completed transcripts from a phone or tablet on the same trusted network.

Arbitrary workspace

Type a directory path and let KAGE validate it locally before scanning sessions or dispatching work.

Task review

Keep dispatched tasks inside the board, then open an inline detail panel only when you need context.

Remote Link direction

Future optional relay work should stay opt-in and preserve the same local-first execution boundary.

One local workspace for agent memory

Each agent keeps its own native session store. KAGE adds discovery, search, conversion, preview, cleanup, replay, and desktop launch controls on top.

Route Use it when
kage c2x Continue a Claude Code session in Codex.
kage x2q Move Codex context into QoderCLI resume format.
kage sessions --since 90d --limit 120 List a bounded recent session window for large local histories.
kage q2v Replay a QoderCLI session as a local HTML story.
kage serve --password 1234 Open the mobile agent monitor and task board for a trusted phone or tablet.
