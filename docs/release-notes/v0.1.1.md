# KAGE v0.1.1

KAGE v0.1.1 updates the macOS desktop app to feel more like a real agent workspace instead of a small session utility.

## What Changed

- The desktop window now opens larger by default.
- The right side is focused on the current session workspace.
- The terminal / ready terminal canvas is now the primary surface.
- Bridge, Replay, and other secondary actions moved into an `Actions` menu.
- Session context, metadata, recent messages, and search matches moved into a collapsible `Session context` area.
- Directory groups now show agent distribution across Claude Code, Codex, and QoderCLI.
- Agent identity is more visible through consistent color and icon badges.

## Download

Download the unsigned macOS DMG:

```text
https://github.com/farmcan/kage/releases/download/v0.1.1/KAGE-0.1.1.dmg
```

If Gatekeeper blocks the first launch, right-click `KAGE.app`, choose `Open`, then confirm.

## CLI Install

```bash
curl -fsSL https://raw.githubusercontent.com/farmcan/kage/main/install.sh | bash
```

## Known Limitations

- The macOS app is still unsigned unless you build with `KAGE_CODESIGN_IDENTITY`.
- The embedded terminal is session-scoped, not a general terminal replacement.
- Release automation still needs a GitHub token with `workflow` scope before it can be committed.
