# KAGE v0.2.0

**Release title:** KAGE v0.2 — Continue a Claude session in Codex without repeating the context

KAGE v0.2 turns cross-agent session memory into a practical workflow: take a useful Claude Code session, convert it into a native Codex session, and continue from the same project context without pasting the transcript or explaining the backstory again.

## Try The Core Workflow

```bash
curl -fsSL https://raw.githubusercontent.com/farmcan/kage/main/install.sh | bash
kage doctor
cd /path/to/your/project
kage c2x
```

KAGE writes a local Codex session and prints the exact command to continue:

```bash
codex resume <generated-session-id>
```

## What v0.2 Adds

- **Native cross-agent continuation.** Claude Code, Codex, and QoderCLI bridges use target-native UUID session ids and preserve source lineage so imported work is easier to identify and resume.
- **Explicit nested-context control.** List or include Claude subagents and QoderCLI sidechains when they matter; they remain excluded by default.
- **One local session layer.** Search, browse, clone, bridge, replay, and safely clean local agent sessions from the CLI or macOS app.
- **A mobile view of local work.** `kage serve` adds a password-capable session monitor, transcript reader, task board, and optional dispatch surface for a trusted LAN.
- **A clearer desktop workflow.** Cached loading, process activity, embedded continuation, and bridge result actions reduce the distance between finding a session and using it.

## Other Useful Routes

```bash
kage x2c  # Codex -> Claude Code
kage x2x  # Clone a Codex session
kage c2c  # Clone a Claude Code session
kage q2x  # QoderCLI -> Codex
```

## Known Limitations

- The macOS DMG is unsigned. If macOS blocks the first launch, right-click `KAGE.app`, choose `Open`, then confirm.
- A bridge carries the visible conversation, project context, and KAGE lineage into a native session file; it cannot transfer opaque model state or an agent's in-memory tool runtime.
- QoderWork is available as a discovery, search, replay, and bridge source, but native QoderWork continue and fork actions remain disabled until a stable resume interface exists.
- Codex nested child-run import is disabled until Codex exposes a stable child-run marker.
- Run `kage serve` only on a trusted network. Use `--password <pin>` for LAN access and add `--read-only` when you do not need dispatch.
